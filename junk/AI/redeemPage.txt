[## Basic ##]
ผลิตหน้า CRM ที่สวยงามสำหรับนำไปใช้งานจริง 
เป็นหน้าย่อยที่จะถูกระบบ pagemanger โหลดไป innerHTML ภายหลัง
ห้ามใส่พื้นหลังเข้ามา เพราะมันมี main page ที่ใส่พื้นหลังเอาไว้แล้ว


[## Language ##]
ดึงภาษาที่ user เลือกไว้เพื่อนำมาแสดงผล
 crmData.language = "En","Th" จุดแสดงภาษา text ทุกจุดในหน้านี้ทำเป็น dict key/value สำหรับ Th : {}  / En : { } แยกเอาไว้ 
 ให้ฉันไปแก้ได้เองในอนาคตง่ายๆ และแต่ละจุดในหน้เว็ปให้เอา key ไปใส่แทนเป็นระบบ language data

[## Script ##]
javascript ห้ามใส่ style ที่ปรับเปลี่ยนความงามใดๆลงไปทั้งสิ้น ให้ template ซ่อนไว้แล้วเปิด/ปิด หรือดึง id มาใช้
เพราะมี style จัดการอยู่แล้ว ฉันไม่ต้องการมานั่งกังวลเรื่อง style ทั้งในไฟล์ css ทั้งฝั่งอยู่ใน js มันควรจัดการที่ css ที่เดียว 

[## CSS Style ##]
ให้ดู Global Override จาก style.css เป็น master style เพื่อให้ทุกหน้าอยู่ใน Theme เดียวกัน
และ css สำหรับจัดการหน้านี้ให้ใส่ไว้ในหน้านี้ได้เลยแต่ให้อ้างอิง master มาจากส่วนกลางเพื่อให้ปรับมูดได้
โดยตอนที่เรายังพัฒนาอยู่ในให้ใส่ style เอาไว้ในหน้า page นี้เลยปรับแต่งให้สวยงามในหน้าเพจนี้
ฉันจะดึง style ทุกหน้าไปรวมกันภายหลังเพื่อให้ปรับได้เองทั้งหมด แต่ให้ออกแบบสำหรับใช้ Global Override ได้เอาไว้เลย
**สำคัญ ห้ามใช้ tailwind ต้องเป็น div + css ธรรมดาเท่านั้น**

[## Service Function ##]
มีฟังชั่นเกี่ยวกับ popup แสดงเตือน หรือ confirm ให้ใช้ของระบบ
PageManager.showConfirm( header, message, image, onYes, onNo );
PageManager.showOk( header, message, image, onOk );
PageManager.showFailed( header, message, image, onOk );
PageManager.showComplete( header, message, image, onOk );
PageManager.showToast( header, icon );
Utility.getAssetPath("pointIcon"); ใช้สำหรับดึง path icon point ของแอฟ

มี loading ของระบบให้ใช้กรณีต้องการรอคอย api ใดๆ
PageManager.showLoading( text );
PageManager.hideLoading(   );

[## Help ##]
ส่วนที่ควรจะตั้งค่าได้เองเช่นรูป / สี / สไตล์ / จำนวน หรืออะไรก็แล้วแต่
ให้ทำเป็น pageSetting = { }; เอาไว้ด้านบน และเขียน comment อธิบายให้ละเอียด

[## Requirement ##]
สร้างหน้าสำหรับ redeem reward ที่สามารถเห็นรายละเอียดของรางวัลได้ครบถ้วน เลือก เพิ่ม/ลด จำนวน
เห็นเรายละเอียดและเงื่อนไขต่างๆ เลือกที่อยู่จัดส่ง / และ confirm มีการตรวจเช็คประเภทรางวัลและจัดการตามเงื่อนไขแต่ละประเภท

เริ่มต้นตอนโหลด page ให้เก็บ rewardId ที่ส่งเข้ามาเอาไว้เพื่อนำไปดึงข้อมูลรางวัลมาแสดงผล
window.onInit = function(payload, response) {
    REWARD_ID = payload.rewardId;
};

[API getReward] ส่ง REWARD_ID ไปดึงค่ามาจาก API นี้
reward = await crmApi.getReward(REWARD_ID);

หน้า interface จะแบ่งเป็น 2 หน้า 
1. information :
- แสดงรุป rewardimage
- ชื่อ / ราคา (points ที่ใช้)
- ประเภทรางวัล rewardType ("physical" | "digital" | "game") :  แสดงเป็นแบชพร้อมไอคอนประเภท
- description / termsConditions :  แสดงเป็นแท็ป กดเพื่อสลับกันแสดง
- oneTimeRedemption --> alreadyRedeemed
- tierRestricted --> requiredTiers
- ปุ่ม เพิ่ม/ลด จำนวน
- ปุ่ม กดต่อไป (แสดงจำนวนรวม points ที่ใช้แลก ราคา x จำนวน)
- ดึง path รูป icon point มาแสดงจาก Utility.getAssetPath("pointIcon");

const canRedeem = Utility.RewardCanRedeem( reward );



2. หน้า final confirm

- เฉพาะ reward.rewardType == "physical" สินค้าที่ต้องทำการ shipping ต้องแสดงส่วนของที่อยู่จัดส่ง
ให้แสดงที่อยู่จัดส่ง ดึงค่า default มาจาก function นี้ หากไม่มี address = null ปุ่มยืนยันการแลกจะไม่สามารถกดได้
let address = Utility.getDefaultAddress();
- มีปุ่มเปลี่ยน/แก้ไข ที่อยู่จัดส่ง หากกดปุ่มนี้ให้จะเปิดหน้า Modal เปิดหน้า address ขึ้นมาให้เลือกที่อยู่จัดส่งใหม่
PageManager.openModal("address",{ mode : "selete" }, function (res)=>{

    if(res.success){
        // ทำการ refresh ที่อยู่จัดส่งที่ user เลือกมาใหม่
        address = res.address;
    }

});
- กรณีเป็นสินค้าที่ไม่ต้องทำการ shipping ให้แสดงข้อความ
 "หมายเหตุ : รางวัลนี้เป็นสินค้าที่ไม่ต้องทำการจัดส่ง" แสดงให้สวยงาม
- สรุปรายการสินค้าที่แลก แบบระเอียด
- ด้านล่าง แสดง จำนวน points final ที่ใข้ และปุ่ม ยืนยันการแลก

ปรับปรุง

- รูปมีขนาดใหญ่เกินไป ทำให้เป็นจตุรัส และกรอบที่สวยงาม
- ด้านบนไม่ต้องมีปุ่ม back เพราะหน้า iframe มีปุ่มปิดของมันอยู่แล้ว 
- ตอนนี้มันโดน mask สีดำบังอยู่ทำให้กดไม่ได้ และไม่สวยงาม ปุ่มปิดของ iframe ต้องอยู่ด้านบนสุด
- ต้องการปรับให้การแสดงผลมันพอดีในหนึ่งหน้าโดยไม่ต้องไถ่สกอลงไป
- นำแต้มที่ใช้แลก ไว้ด้านขวา ตรงข้ามกับ ชื่อ และ ประเภท น่าจะดีกว่า
- ส่วนของรายละเอียด และ เงื่อนไข ล็อกความสูงเอาไว้ เวลากดสลับไปมา
มันต้องไม่เปลี่ยนแปลงขนาด และ ทำให้มันไถ่สกอดูข้อมูลได้สะดวก
- ปุ่มกดเพิ่ม/ลดจำนวน และ ปุ่มถัดไปให้อยู่บรรทัดเดียวกัน ย้ายปุ่มเพิ่มลดมาไว้ซ้ายล่าง
และนำแต้มที่ใช้แลกไปไว้ในปุ่มด้านขวาสุดเลย ให้ปุ่มแสดงจำนวนแต้มไปเลย เพื่อประหยัดพื้นที่

- หน้ายืนยัน ให้แสดงข้อมูลสินค้าที่ต้องการแลกให้ละเอียดขึ้นอีก
ใส่รูป และ ประเภทสินค้า คำอธิยายหากยาวไปให้ตัดทิ้งเลยและเติม ... ด้านหลัง

- แสดง icon points ไว้ด้วยตอนจำนวน point สุดท้ายก่อนยืนยันการแลก

- การเปลี่ยนที่อยู่จัดส่ง ตอนกดปุ่มเปลี่ยน/แก้ไข ที่อยู่จัดส่ง 
หากกดปุ่มนี้ให้จะเปิดหน้า Modal เปิดหน้า address ขึ้นมาให้เลือกที่อยู่จัดส่งใหม่
เปลี่ยนการรับค่าจาก page address ไปใช้แบบที่ฉันเขียนไว้นี้
PageManager.openModal("address",{ mode : "selete" }, function (res){
    if(res.success){
        // ทำการ refresh ที่อยู่จัดส่งที่ user เลือกมาใหม่
        address = res.address;
    }
}); 

- กรณีไม่มีที่อยู่จัดส่ง แสดงข้อความ
ปรับปรุงให้สวยงามขึ้น เติม icon เข้าไปให้ดูดี


- หน้ายืนยัน ทั้งสองโซน "ที่อยู่จัดส่ง" และ "ของรางวัล"
ให้เติม header และ icon ด้านหน้า text ไว้ทั้งสอง panel เพื่อแยกแต่ละโซนให้ชัดเจน
และทำให้ โซนที่อยู่จัดส่งมีบอกชัดเจนว่าเป็นโซนที่อยู่จัดส่ง ตอนนี้มันโผล่มาก็หมายเหตุเลย
ทำให้ไม่รู้ว่ามันคือเรื่องอะไร

[หน้ายืนยัน]
1.ตรงจุดแสดงรางวัล ให้ icon กับจำนวนคะแนนอยู่บรรทัดเดียวกัน icon อยู่ด้านหน้าตัวเลข และตามด้วยคะแนน
2.  panel แสดงผลรวม ให้ total point: อยู่ด้านซ้าย และ Point ที่ใช้อยู่ด้านขวา บรรทัดเดียวกัน
ย้าย icon มาด้านหน้า และตามหลังด้วยคำว่า Pts หรือ คะแนน
3.  panel แสดงผลรวม ย้ายมาชิดกรอบล่าง ตอนนี้มันขึ้นมาทับตัวรางวัล มันบังการแสดงผล
