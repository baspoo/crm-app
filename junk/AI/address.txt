นี้คือหน้า page เดิมใช้สำหรับเปิดใน iframe ที่ใช้งานไม่ได้แล้ว
ให้เปลี่ยนเป็นระบบใหม่ทั้งหมด  สำหรับนำไป inner เข้าหน้าหลักเหมือนกับหน้าอื่นๆ ในโปรเจ็คใหม่นี้ 
วิเคาะห์การทำงานเดิมของหน้านี้และเปลี่ยน โครงสร้างให้สามารภทำงานได้เหมือนเดิม โดยใช้ระบบเรียก api ใหม่
เปลี่ยนโครงสร้าง css สร้าง pageSetting สำหรับตั้งค่า และอื่นๆที่จำเป็น
ยกเลิกการใช้ tailwind ใน code ให้เขียนแปลงเป็น css 100%

ใช้ Utility.getAssetPath("xxxxxx"); สำหรับ load image path โดยใช้ key เดิม


รับค่าเข้าด้วย window.onInit แบบนี้ และเก็บ response เอาไว้เป็น ref สำหรับส่งค่ากลับ
window.onInit = function(payload, response) {
    console.log("ได้รับ Payload:", payload);
    // เซ็ตข้อมูลกลับได้ผ่าน response
    response.status = "ok";
};




[## Basic ##]
ผลิตหน้า CRM ที่สวยงามสำหรับนำไปใช้งานจริง 
เป็นหน้าย่อยที่จะถูกระบบ pagemanger โหลดไป innerHTML ภายหลัง
ห้ามใส่พื้นหลังเข้ามา เพราะมันมี main page ที่ใส่พื้นหลังเอาไว้แล้ว


[## Language ##]
ดึงภาษาที่ user เลือกไว้เพื่อนำมาแสดงผล
 crmData.language = "En","Th" จุดแสดงภาษา text ทุกจุดในหน้านี้ทำเป็น dict key/value สำหรับ Th : {}  / En : { } แยกเอาไว้ 
 ให้ฉันไปแก้ได้เองในอนาคตง่ายๆ และแต่ละจุดในหน้เว็ปให้เอา key ไปใส่แทนเป็นระบบ language data

[## Script ##]
javascript ห้ามใส่ style ที่ปรับเปลี่ยนความงามใดๆลงไปทั้งสิ้น ให้ template ซ่อนไว้แล้วเปิด/ปิด หรือดึง id มาใช้
เพราะมี style จัดการอยู่แล้ว ฉันไม่ต้องการมานั่งกังวลเรื่อง style ทั้งในไฟล์ css ทั้งฝั่งอยู่ใน js มันควรจัดการที่ css ที่เดียว 

[## CSS Style ##]
ให้ดู Global Override จาก style.css เป็น master style เพื่อให้ทุกหน้าอยู่ใน Theme เดียวกัน
และ css สำหรับจัดการหน้านี้ให้ใส่ไว้ในหน้านี้ได้เลยแต่ให้อ้างอิง master มาจากส่วนกลางเพื่อให้ปรับมูดได้
โดยตอนที่เรายังพัฒนาอยู่ในให้ใส่ style เอาไว้ในหน้า page นี้เลยปรับแต่งให้สวยงามในหน้าเพจนี้
ฉันจะดึง style ทุกหน้าไปรวมกันภายหลังเพื่อให้ปรับได้เองทั้งหมด แต่ให้ออกแบบสำหรับใช้ Global Override ได้เอาไว้เลย
**สำคัญ ห้ามใช้ tailwind ต้องเป็น div + css ธรรมดาเท่านั้น**

[## Requirement ##]
สร้างหน้า สำหรับ เพิ่ม/ลบ แก้ไข ที่อยู่ของ user ในแอป crm ของฉัน
หน้าเดิมที่ฉันทำไวจะเป็นเพียงการแก้ไขที่อยู่เท่านั้น ตอนนี้ฉันต้องการทำให้มันแสดงรายการที่อยู่ที่มี และ สามารถ เพิ่ม/ลบ หรือ แก้ไขในรายการนั้นๆได้
เมื่อกดรายการใดๆ จะพาไปเปิดหน้า edit คล้ายๆ code เดิมของฉันที่ได้ส่งไปให้
ออกแบบ UI ให้ดูทันสมัย สวยงาม และจัดวางปุ่ม เพิ่ม ลบ แก้ไข ให้ดูมืออาชีพ เข้าใจง่าย เติม icon ในจุดต่างๆเพื่อช่วยอธิบายในการใช้งาน


การเปิดหน้านี้ จะมีลูกเล่นนิดหน่อยเพื่อสามารถใช้ซ้ำได้ในกรณีเปิดเข้ามาเพื่อเลือกที่อยู่นำไปใช้งาน
โดยจะแบ่งเป็น 2 mode คือ edit และ select
edit : คือโหมดปกติที่จะเข้ามาเพิ่ม ลด แก้ไข ที่อยู่ของ user
select : คือโหมดที่จะเข้ามาเพื่อเลือกที่อยู่สำหรับนำไปใช้แต่ละ feature เปิดมาเลือกรายการที่ต้องการและส่ง address id กลับไป

window.onInit = function(payload, response) {
   
    // จะส่ง mode เข้ามาทาง init หากไม่ส่งมาคือ edit เป็น default  ["edit" , "select"]
    MODE = payload.mode || "edit";

    // หากเป็น select mode ให้ส่งค่ากลับเข้าไปผ่าน response ที่ส่งเข้ามา
    response.selete = addressId;

    // ส่ง success เป็น true เสมอ หากโหมด edit มีการแก้ไข เกิดขึ้น หรือ มีการกดเลือกเพื่อส่งกลับในโหมด select
    response.success = true;

};

const addresses = crmData.crmUser.userAddresses;

นี้คือ function สำหรับ ใช้งาน api function ต่างๆที่ฉันเตรียมไว้แล้ว

[API Create New Address]
สร้างใหม่ไม่มี id
const newAddress = {
    contactPerson,
    addressLine1,
    addressLine2,
    district,
    postalCode,
    contactPhone,
    ..
};
await crmApi.createAddress(newAddress);

[API Update Address]
หากเป็นการอัพเดทต้องแนบ id ไปด้วยเสมอ
const updateAddress = {
    id,
    contactPerson,
    addressLine1,
    addressLine2,
    district,
    postalCode,
    contactPhone,
    ..
};
await crmApi.updateAddress(updateAddress);


[API Set Default Address] สำหรับปุ่มตั้งค่าให้ที่อยู่นี้เป็นที่อยู่เริ่มต้นสำหรับใช้งาน แอป CRM 
มักใช้ในกรณี redeem reward จะจัดส่งไปที่อยู่ default เสมอ ยกเว้นจะกดเข้ามา selete รายการที่ต้องการเอง
await crmApi.setDefaultAddress(address.id);

[API Remove Address] สำหรับลบที่อยู่
await crmApi.onRemoveAddress(address.id);
